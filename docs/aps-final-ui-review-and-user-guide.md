# APS Final UI Review and User Guide

วันที่จัดทำ: 05/06/2026  
ใช้สำหรับเตรียมพรีเซนต์ Final วันที่ 10/06/2026

## 1. สรุปภาพรวมระบบแบบเข้าใจง่าย

ระบบ APS (Academic Planning Support) คือเว็บแอปสำหรับช่วยนักศึกษาวางแผนการเรียนตามหลักสูตร โดยนำข้อมูลผลการเรียนของนักศึกษาไปเทียบกับข้อมูลหลักสูตร รายวิชา วิชาบังคับก่อน แผนการเรียน และวิชาที่เปิดสอนในแต่ละเทอม เพื่อแสดงผลว่า

- นักศึกษามีสถานะการเรียนโดยรวมเป็นอย่างไร
- เหลือหน่วยกิตและรายวิชาใดบ้าง
- คาดว่าจะจบได้ในปี/เทอมใด
- รายวิชาใดกำลังขวางการจบเพราะติด prerequisite หรือผลการเรียน F/W
- หากถอนรายวิชา ลงรายวิชาเพิ่ม หรือไม่ผ่านรายวิชา จะกระทบแผนจบอย่างไร
- ผู้ดูแลระบบสามารถจัดการข้อมูลหลักสูตรผ่านหน้า UI หรือ import CSV ได้

ดังนั้นประโยคหลักของระบบไม่ควรสื่อว่าเป็น "ระบบอัปโหลด transcript" เท่านั้น แต่ควรสื่อว่าเป็น "ระบบช่วยวิเคราะห์และวางแผนการเรียนจนจบหลักสูตร"

## 2. ปัญหา UI/UX ที่ควรแก้ก่อนพรีเซนต์

### 2.1 ปัญหาหลัก

1. ระบบมีฟังก์ชันเยอะ แต่ผู้ใช้ยังไม่รู้ว่าต้องเริ่มตรงไหน
2. หน้า student dashboard มีข้อมูลหลายชุด จึงควรจัดตามคำถามของนักศึกษา ไม่ใช่ตามชื่อ module
3. ปุ่มอัปโหลด/แก้ข้อมูลผลการเรียนต้องเห็นชัด เพราะเป็นจุดเริ่มต้นของการวิเคราะห์
4. หน้า admin ไม่ควรให้ความรู้สึกเหมือน CRUD ดิบ ควรเล่าเป็นลำดับงาน: เตรียมหลักสูตร -> รายวิชา -> prerequisite -> แผนรายเทอม -> วิชาที่เปิดสอน -> ตรวจความพร้อม
5. คำบนปุ่มและหัวข้อควรใช้ภาษาเดียวกันทั้งระบบ เช่น "คาดการณ์วันจบ", "จำลองสถานการณ์", "จัดการข้อมูลผลการเรียน"
6. ควรลดความรู้สึก AI-made โดยลด card ซ้ำ ๆ, ลด gradient ที่ไม่จำเป็น, ใช้ layout แบบ dashboard จริง, และเน้น hierarchy ที่อ่านเร็ว

### 2.2 สิ่งที่ควรปรับแบบเร่งด่วน

ลำดับ 1: ทำหน้าแรกให้ตอบ 3 คำถามทันที

- ระบบนี้ช่วยใคร
- ต้องเริ่มใช้งานอย่างไร
- หลังใช้งานจะได้ผลลัพธ์อะไร

ลำดับ 2: ทำ Student Dashboard ให้เป็น workflow

- แถบบน: สถานะนักศึกษา, GPAX, หน่วยกิตสะสม, คาดการณ์วันจบ
- กล่องเริ่มต้น: "จัดการข้อมูลผลการเรียน" พร้อมปุ่มอัปโหลด PDF / แก้รายวิชาเอง
- เมนูหลัก: ภาพรวม, คาดการณ์วันจบ, แผน 8 ปี, จำลองสถานการณ์
- ส่วนท้าย: ข้อควรระวังและคำแนะนำ

ลำดับ 3: ทำ Admin ให้เป็น setup checklist

- ตรวจความพร้อมข้อมูล
- import CSV template
- จัดการหลักสูตร
- จัดการรายวิชา
- จัดการวิชาบังคับก่อน
- จัดการแผนรายเทอม
- จัดการวิชาที่เปิดสอน

ลำดับ 4: ใส่ข้อความช่วยอธิบายเฉพาะจุดที่ผู้ใช้ลังเล

- ก่อนอัปโหลด PDF: "หากระบบอ่านไม่ครบ สามารถแก้ไขรายวิชาก่อนยืนยันได้"
- ก่อนดูผลคาดการณ์: "ผลลัพธ์ขึ้นอยู่กับข้อมูลหลักสูตรและวิชาที่เปิดสอน"
- ก่อนจำลองสถานการณ์: "การจำลองไม่บันทึกทับข้อมูลจริง"

## 3. Requirement ที่ควรใช้เป็นฐานในการเล่าระบบ

### 3.1 ผู้ใช้ฝั่งนักศึกษา

- เข้าสู่ระบบด้วยบัญชีมหาวิทยาลัย
- เลือก/ตรวจสอบหลักสูตรของตนเอง
- อัปโหลดไฟล์ PDF ผลการเรียน หรือเพิ่มรายวิชาด้วยตนเอง
- ตรวจสอบรายวิชาที่ระบบอ่านได้ก่อนยืนยัน
- ดูสถานะ GPAX, GPA ล่าสุด, หน่วยกิตสะสม และสถานะติดโปร
- ดูรายวิชาที่ผ่านแล้ว ยังไม่เรียน ไม่ผ่าน ถอน หรือไม่นับหน่วยกิต
- ดูแผนการเรียนรายเทอมและคาดการณ์วันจบ
- ดูรายวิชาที่ขวางการจบจาก prerequisite
- จำลองสถานการณ์ เช่น ถอนรายวิชา ลงเพิ่ม หรือไม่ผ่านรายวิชา
- ได้คำแนะนำว่าควรทำอะไรต่อ

### 3.2 ผู้ใช้ฝั่งผู้ดูแลระบบ

- เข้าสู่ระบบด้วยสิทธิ์ admin
- ตรวจความครบถ้วนของข้อมูลก่อนเผยแพร่
- จัดการข้อมูลหลักสูตร
- จัดการโครงสร้างหมวดวิชาและหน่วยกิตขั้นต่ำ
- จัดการรายวิชา
- จัดการ prerequisite / corequisite
- จัดการแผนการเรียนรายปี/รายเทอม
- จัดการวิชาที่เปิดสอนในแต่ละเทอม
- import ข้อมูลผ่าน CSV template
- ตรวจสอบข้อผิดพลาดของข้อมูล เช่น รายวิชาไม่มีเทอมเปิด หรือ prerequisite หาย

## 4. คู่มือการใช้งานสำหรับนักศึกษา

### ขั้นตอนที่ 1: เข้าสู่ระบบ

1. เปิดหน้าแรกของระบบ APS
2. กดปุ่มเข้าสู่ระบบด้วยบัญชี Google ของมหาวิทยาลัย
3. ระบบตรวจสอบสิทธิ์ผู้ใช้
4. เมื่อเข้าสู่ระบบสำเร็จ ระบบพาไปยัง Student Dashboard

ผลลัพธ์ที่ควรเห็น:

- ชื่อระบบ
- สถานะนักศึกษา
- GPAX/GPA ล่าสุด
- หน่วยกิตสะสม
- ปุ่มไปจัดการข้อมูลผลการเรียน

### ขั้นตอนที่ 2: ตั้งค่าหลักสูตร

1. ไปที่หน้า "จัดการข้อมูลผลการเรียน"
2. ตรวจสอบหลักสูตรที่ระบบใช้วิเคราะห์ เช่น CS2565 หรือ IT2565
3. ระบุรหัสนักศึกษาและ track หากระบบรองรับ
4. บันทึกข้อมูล

ข้อควรระวัง:

- หากเลือกหลักสูตรผิด ผลคาดการณ์วันจบและรายวิชาคงเหลือจะผิดตาม
- ควรตรวจปีหลักสูตรให้ตรงกับหลักสูตรจริงของนักศึกษา

### ขั้นตอนที่ 3: นำเข้าข้อมูลผลการเรียน

1. ไปที่หน้า "จัดการข้อมูลผลการเรียน"
2. เลือกไฟล์ PDF transcript
3. ให้ระบบอ่านรายวิชา เกรด หน่วยกิต ปีการศึกษา และเทอม
4. ตรวจตาราง preview ก่อนยืนยัน
5. หากข้อมูลอ่านไม่ครบ ให้แก้ไขหรือเพิ่มรายวิชาด้วยตนเอง
6. กดยืนยันข้อมูล

ผลลัพธ์ที่ควรเห็น:

- รายวิชาที่อ่านได้จาก PDF
- สรุป GPA/GPAX รายเทอม
- warning หากข้อมูลบางส่วนอ่านไม่ครบ
- ปุ่มยืนยันข้อมูลก่อนนำไปวิเคราะห์

ข้อควรระวัง:

- PDF บางรูปแบบอาจอ่านไม่ครบ 100%
- ไฟล์สแกนภาพอาจต้องใช้ OCR และอาจเกิดความคลาดเคลื่อน
- ต้องตรวจข้อมูลก่อนยืนยันทุกครั้ง

### ขั้นตอนที่ 4: ดูภาพรวมผลการเรียน

1. กลับไปที่ Student Dashboard
2. เลือกเมนู "ภาพรวม"
3. ตรวจสถานะ GPAX, GPA ล่าสุด, หน่วยกิตสะสม และสถานะความเสี่ยง
4. อ่านคำแนะนำที่ระบบแสดง

ระบบควรช่วยตอบคำถาม:

- ตอนนี้สถานะการเรียนเป็นปกติหรือเสี่ยง
- หน่วยกิตสะสมถึงเกณฑ์หรือยัง
- มีหมวดวิชาใดที่ยังขาดอยู่
- ควรแก้ปัญหาใดก่อน

### ขั้นตอนที่ 5: ดูคาดการณ์วันจบ

1. เลือกเมนู "คาดการณ์วันจบ"
2. ระบบแสดงแผนรายเทอมจากปัจจุบันจนจบ
3. ตรวจรายวิชาที่ระบบเสนอให้ลงในแต่ละเทอม
4. ดูว่าคาดว่าจะจบในปี/เทอมใด

ข้อควรระวัง:

- ผลคาดการณ์ขึ้นอยู่กับข้อมูลวิชาที่เปิดสอน
- หากข้อมูลวิชาเปิดสอนไม่ครบ ระบบอาจวางแผนไม่ได้บางรายวิชา
- รายวิชาที่ติด prerequisite อาจทำให้วันจบเลื่อนออกไป

### ขั้นตอนที่ 6: ดูแผนการเรียน 8 ปี

1. เลือกเมนู "แผนผังการเรียนรายเทอม"
2. ดูรายวิชาตามปีและเทอม
3. ตรวจสีหรือสถานะของรายวิชา เช่น ผ่านแล้ว, วางแผนไว้, ไม่ผ่าน, ถอน, ขวางการจบ
4. ใช้หน้านี้เพื่ออธิบายภาพรวมแผนการเรียนระหว่างพรีเซนต์

จุดที่ควรเน้นตอนพรีเซนต์:

- แผนนี้ช่วยให้นักศึกษาเห็นลำดับวิชาต่อเนื่อง
- รายวิชาที่เป็น prerequisite สำคัญจะเห็นผลกระทบชัดขึ้น
- นักศึกษาสามารถรู้ได้ว่าวิชาใดควรลงก่อน

### ขั้นตอนที่ 7: จำลองสถานการณ์

1. เลือกเมนู "จำลองสถานการณ์"
2. เลือกกรณีที่ต้องการจำลอง เช่น ถอนรายวิชา ลงรายวิชาเพิ่ม หรือไม่ผ่านรายวิชา
3. เลือกปีการศึกษาและเทอม
4. กดคำนวณ
5. เปรียบเทียบแผนเดิมกับแผนจำลอง

ผลลัพธ์ที่ควรเห็น:

- วันจบเดิมเทียบกับวันจบหลังจำลอง
- รายวิชาที่ถูกปลดล็อก
- รายวิชาที่ถูกขวางเพิ่ม
- จำนวนเทอมที่อาจล่าช้า
- คำอธิบายสรุปผลกระทบ

ข้อควรระวัง:

- การจำลองไม่ใช่การบันทึกข้อมูลจริง
- ใช้เพื่อประกอบการตัดสินใจก่อนลงทะเบียน
- ผลลัพธ์ควรตรวจสอบกับอาจารย์ที่ปรึกษาอีกครั้ง

## 5. คู่มือการใช้งานสำหรับผู้ดูแลระบบ

### ขั้นตอนที่ 1: เข้าหน้า Admin

1. เข้าสู่ระบบด้วยบัญชีที่มีสิทธิ์ admin
2. เปิดหน้า Admin Control Panel
3. เริ่มจากเมนู "ภาพรวมความพร้อมก่อนเผยแพร่"

หน้าภาพรวมควรบอก:

- ข้อมูลหลักสูตรครบหรือไม่
- รายวิชาครบหรือไม่
- prerequisite ครบหรือไม่
- แผนรายเทอมครบหรือไม่
- วิชาที่เปิดสอนครบหรือไม่

### ขั้นตอนที่ 2: Import CSV Template

1. ดาวน์โหลดหรือเตรียมไฟล์ CSV ตาม template
2. เลือกประเภทข้อมูลที่ต้องการ import
3. อัปโหลดไฟล์
4. ตรวจผลการ import
5. แก้ไขข้อผิดพลาดหากมี

ข้อมูลที่ควรรองรับ:

- หลักสูตร
- โครงสร้างหลักสูตร
- รายวิชา
- วิชาบังคับก่อน
- แผนรายเทอม
- วิชาที่เปิดสอน

### ขั้นตอนที่ 3: จัดการหลักสูตร

1. เลือกเมนู "หลักสูตร"
2. เพิ่มหรือแก้ไขรหัสหลักสูตร ชื่อหลักสูตร ปีหลักสูตร และหน่วยกิตขั้นต่ำ
3. บันทึกข้อมูล

ตัวอย่างข้อมูลสำคัญ:

- program code
- ชื่อหลักสูตรภาษาไทย/อังกฤษ
- ปีหลักสูตร
- หน่วยกิตขั้นต่ำ
- เกณฑ์ GPAX สำหรับเกียรตินิยม

### ขั้นตอนที่ 4: จัดการรายวิชาและ prerequisite

1. เลือกเมนู "รายวิชา"
2. เพิ่มรหัสวิชา ชื่อวิชา หน่วยกิต และหมวดวิชา
3. เลือกเมนู "วิชาบังคับก่อน"
4. กำหนดความสัมพันธ์ระหว่างวิชาต้นทางและวิชาต่อเนื่อง
5. ระบุว่าเป็น prerequisite หรือ corequisite

ข้อควรระวัง:

- รหัสวิชาต้องตรงกับ transcript
- หาก prerequisite หาย ระบบจะวิเคราะห์ผลกระทบผิด
- หากหมวดวิชาไม่ตรง ระบบสรุปหน่วยกิตตามหมวดผิด

### ขั้นตอนที่ 5: จัดการแผนรายเทอมและวิชาที่เปิดสอน

1. เลือกเมนู "แผนรายเทอม"
2. ระบุว่ารายวิชาใดควรเรียนในปีและเทอมใด
3. เลือกเมนู "วิชาเปิด"
4. ระบุว่าวิชานั้นเปิดสอนในเทอม 1, เทอม 2 หรือภาคฤดูร้อน
5. กลับไปตรวจความพร้อมอีกครั้ง

ข้อควรระวัง:

- แผนรายเทอมใช้สร้าง roadmap
- วิชาที่เปิดสอนใช้คาดการณ์วันจบ
- ถ้ารายวิชาจำเป็นไม่มีเทอมเปิด ระบบอาจวางแผนจบไม่ได้

## 6. Flow ที่ควรใช้ตอนพรีเซนต์

### Demo Flow ฝั่งนักศึกษา

1. เปิดหน้าแรกและอธิบายว่า APS ช่วยวางแผนการเรียนจนจบหลักสูตร
2. เข้าสู่ระบบเป็นนักศึกษา

## 8. คำอธิบายระบบแบบพูดหน้าห้อง

ระบบ APS เป็นระบบสนับสนุนการวางแผนการเรียนสำหรับนักศึกษา โดยใช้ข้อมูลผลการเรียนของนักศึกษาเทียบกับข้อมูลหลักสูตร รายวิชา วิชาบังคับก่อน แผนการเรียน และรายวิชาที่เปิดสอนในแต่ละภาคการศึกษา จากนั้นระบบจะแสดงสถานะการเรียน หน่วยกิตที่สะสม รายวิชาที่ยังขาด รายวิชาที่อาจขวางการจบ และคาดการณ์ภาคการศึกษาที่จะสำเร็จการศึกษา นอกจากนี้ยังมีส่วนจำลองสถานการณ์ เพื่อให้นักศึกษาทดลองดูผลกระทบก่อนตัดสินใจถอนรายวิชา ลงรายวิชาเพิ่ม หรือประเมินกรณีไม่ผ่านรายวิชา ส่วนผู้ดูแลระบบสามารถจัดการข้อมูลหลักสูตรและข้อมูลประกอบการวิเคราะห์ผ่านหน้า admin หรือ CSV template ได้โดยไม่ต้องแก้ไขโค้ดโดยตรง

## 9. ข้อจำกัดที่ควรพูดอย่างมั่นใจ

- การอ่าน PDF transcript ยังไม่สามารถรับประกันความถูกต้อง 100% กับทุกไฟล์
- ระบบจึงออกแบบให้มีขั้นตอน preview และแก้ไขข้อมูลก่อนยืนยัน
- ผลคาดการณ์ขึ้นอยู่กับความครบถ้วนของข้อมูลหลักสูตร prerequisite และวิชาที่เปิดสอน
- ข้อมูลตัวอย่างควรตรวจสอบกับหลักสูตรจริงก่อนนำไปใช้จริง
- ระบบยังไม่ครอบคลุมทุกกรณีพิเศษ เช่น การเทียบโอน รายวิชาแทนกัน หรือเงื่อนไขเฉพาะรายบุคคล
