# APS Golden Cases

เอกสารนี้คือ expected result สำหรับใช้ตรวจ logic การคาดการณ์จบของ APS จาก transcript ตัวอย่างก่อน final thesis

| Case | Program | Track | Expected result |
| --- | --- | --- | --- |
| IT_Sand | IT2565 | Research | ผ่านครบตามสายโครงงานวิจัย และ REG อนุมัติปริญญา เทอม 2/2568 |
| CS_F | CS2565 | Research | ยื่นจบแล้ว รอโครงงานวิจัย 2 (`517494`) เทอม 3/2568 |
| CS_Nook | CS2565 | Research | ยื่นจบแล้ว รอโครงงานวิจัย 2 (`517494`) เทอม 3/2568 |
| CS_Golf | CS2565 | Research | ยื่นจบแล้ว รอโครงงานวิจัย 2 (`517494`) เทอม 3/2568 |
| CS_Neoi | CS2565 | Coop | ยื่นจบแล้ว เรียนครบตามแผนสหกิจ แต่รอเกรด I ของ `517497` |
| CS_Minie | CS2565 | Coop | ยื่นจบแล้ว เรียนครบตามแผนสหกิจ แต่รอเกรด I ของ `517497` |
| CS_Moss | CS2565 | Research | ยังไม่ยื่นจบ เพราะผ่านโครงงานวิจัยแล้วแต่รายวิชาปกติยังไม่ครบ |

## Rule Notes

- `I` ไม่ใช่ `F` และต้องแสดงเป็นสถานะรอผลเกรด
- เคสที่ยื่นจบแล้วแต่มีวิชารอผลต้องอยู่ในกลุ่ม `applied_pending_result`
- เคสสหกิจต้องตรวจ `517497` แยกจากเงื่อนไขโครงงานวิจัย
- เคส Research ที่รอโครงงานวิจัย 2 ต้องผูกกับ `517494`
- ระบบห้ามสรุปสถานะ REG เองจากผลการเรียน ต้องใช้ `regStatus` แยกต่างหาก

ตรวจ fixture ได้ด้วย:

```bash
npm run test:golden
```
