## Grade Policy ที่ควรรองรับ

- `A` ถึง `D` นับเป็นผ่านและนับหน่วยกิต
- `F` ไม่ผ่านและนับเป็น attempt ใน GPAX
- `W` ถอนรายวิชา ไม่ควรนับ GPAX หรือหน่วยกิต
- `I` คือรอเกรด ต้องแสดงเป็น “รอเกรด” ไม่ใช่ “ตก” และยังไม่ควรนับว่าผ่านจนกว่าเกรดจริงออก
- `S` นับผ่านแบบไม่คิด grade point
- `S*` ใช้กับบางกรณีที่ผ่าน/รับทราบผลแต่ไม่ควรนับหน่วยกิตซ้ำ
- `U` ไม่ผ่านแบบไม่คิด grade point

## แนวทางรองรับหลักสูตรอื่นในอนาคต

- แยกตัวตนของรายวิชาเป็น `programCode + courseCode` ไม่ใช้ course code อย่างเดียว
- เก็บ study plan แยกตามหลักสูตร รุ่นปี และ track เช่น research หรือ coop
- ให้ import หลักสูตรจาก CSV/Excel ได้ โดยมีคอลัมน์ program, year, semester, courseCode, credits, category, track
- แยกข้อมูล course กลางออกจากข้อมูลที่ขึ้นกับหลักสูตร เช่น category และเทอมที่แนะนำให้เรียน
- ทำ validation หลัง import ว่าวิชาบังคับครบตามหน่วยกิตขั้นต่ำของหลักสูตร
- ทำหน้าจอ admin แบบ checklist: ตรวจหลักสูตร -> ตรวจรายวิชา -> ตรวจ prerequisite -> ตรวจแผนรายเทอม -> ตรวจวิชาเปิดสอน
