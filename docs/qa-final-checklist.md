## เกณฑ์ผ่านหลัก

ระบบถือว่าพร้อมสอบเมื่อผ่านครบทุกข้อด้านล่าง:

- นำเข้าข้อมูลนักศึกษาได้
- แก้ข้อมูล transcript ก่อนบันทึกได้
- forecast แสดงแผนจบเป็นรายเทอมได้
- what-if เทียบแผนเดิมกับแผนจำลองได้
- admin ปรับข้อมูลหลังบ้านได้โดยไม่แก้ JSON
- UI อ่านง่าย เป็นภาษาไทย และไม่ทำให้ผู้ใช้เข้าใจว่าระบบมีแค่ upload transcript

## เกณฑ์ตัดสินก่อนสอบ

- ถ้า CS_F, CS_Neoi, CS_Minie อ่านได้และบันทึกได้ ถือว่า text-based transcript path ผ่าน
- ถ้า CS_Nook และ IT_Sand อ่านไม่ได้ แต่ผู้ใช้เพิ่ม/แก้ข้อมูลเองจนบันทึกและวิเคราะห์ต่อได้ ถือว่า intake path ผ่าน เพราะระบบไม่ผูกกับ OCR
- ถ้า forecast และ what-if ใช้ข้อมูลที่ admin กรอก/นำเข้าจาก DB ได้ ถือว่า analysis engine ผ่าน
- ถ้า admin สามารถปรับหลักสูตร รายวิชา prerequisite study plan และวิชาเปิดผ่าน UI/CSV ได้ ถือว่า admin UX ผ่าน
- ถ้า student เห็นสถานะ แผนจบ แผนรายเทอม วิชา block และ what-if ก่อน upload tools ถือว่า dashboard UX ผ่าน
